조현근 "도입 전에는 운용수익 가정 기반의 LAT 단순 비교였습니다. 도입 후엔 CSM 기반으로 상품별 수익성을 다시 산정해야 했고, 저희 팀은 변액·종신 비중을 재조정하는 의사결정을 내렸습니다. 특히 무배당종신은 CSM 마진이 기존 추정 대비 18% 줄어들었고, 이에 사업비 구조 자체를 재설계해 첫해 사업비를 12% 절감하는 방향으로 갔습니다."

업무 부여 로드맵 (Onboarding)

디지털 IT 기획팀 합류 후 6개월 권장 흐름입니다. 신입 입사자 특성상 학습·관계 형성·작은 완결 사이클의 반복에 비중을 두었습니다.

PHASE 1 1~2개월

ABL 시스템·도메인 학습 + 멘토 안착

ABL의 IT 시스템(상품계·계약계·정산계) 구조와 보험 도메인 기본 지식을 학습하는 단계. 전담 멘토와의 신뢰 형성이 가장 중요한 시기로, 업무보다 학습·관계 형성에 비중을 둡니다.

학습ABL IT 시스템 구조 학습, 보험 도메인 입문, 사내 코딩 컨벤션 숙지
멘토링전담 멘토 주 2회 30분 1:1, 멘토 업무 쉐도잉

체크포인트매주 멘토 + 팀장 1:1, 8주차 종합 리뷰
PHASE 2 3~4개월

첫 완결 산출물 + 본부 공유

본인 동기 2순위인 "완결된 산출물"을 충족시키는 단계. 작은 단위의 업무 1건을 처음부터 끝까지 책임지게 하고, 본부에 공유하는 자리를 마련합니다.

목표작은 업무 1건(예: 내부 자동화 도구) 완수, 본부 짧은 발표
기대 산출물완성 산출물 1건, 본인 작성 1-pager, 회고 노트
업무 비중60% — 첫 책임 업무 + 멘토 협업 유지
체크포인트중간 리뷰 1회, 산출물 완성 직후 인정 피드백 즉시 전달
PHASE 3 5~6개월

정규 프로젝트 합류 + 1년차 목표 합의

팀의 정규 프로젝트에 한 명의 팀원으로 합류시키는 단계. 6개월차 종합 면담에서 1년차 시점에 어떤 영역을 본인 강점으로 만들지 합의합니다.

목표정규 프로젝트 합류, 본인 명의의 작은 결정 권한 1건 부여
기대 산출물프로젝트 산출물 일부 책임, 6개월 회고 발표
업무 비중100% — 멘토 의존도 점진적 감소, 독립성 점차 확보
